> 版本：Ver.2.1  
> 生效日期：2026年5月29日  
> 制定機關：中央治理委員會

---

## 一、總則

後勤部為 TJSCODE 系統的**倉庫維護及知識管理機構**，負責階段 9（後勤部歸檔及補丁更新）的全部工作。後勤部的核心任務是將每一場賽事的教訓沉澱為補丁，將每一次回測的數據歸檔為系統的長期記憶，並維護 GitHub 倉庫的完整性。

**核心使命**：補丁在，紀律在，火種不滅。

## 三、職責

### 3.1 階段 9：後勤部歸檔及補丁更新

1. **接收賽後檢討**：從戰略部或稽查部接收賽後檢討報告。
2. **提煉補丁**：若實戰中發現流程漏洞或新規律，撰寫補丁提案。
3. **補丁審批**：Peter 審議補丁條款，確保不違反憲法鐵律。
4. **補丁發布**：Gary 批准後，更新 `rules/patches.md` 並發布。
5. **歸檔**：Wendy 將本場所有文件（預測記錄、賽果、財務結算、檢討）歸檔至 `data/archive/`。
6. **倉庫更新**：將所有更新推送至 GitHub 倉庫。

### 3.2 補丁管理

- 補丁提案須包含：觸發條件、強制程序、判例。
- 補丁分為三級：憲法級、強制級、條件級。
- 補丁之新增、修訂、廢止須經稽查部審查、中央治理委員會批准。

## 四、倉庫維護規範

1. 所有檔案須使用 Markdown 格式。
2. 檔案命名遵循現有規範（小寫字母，底線分隔）。
3. 每次更新須在檔案頭部的版本歷史中記錄。
4. 重大更新後須在 `README.md` 中更新版本號及回測成績。

## 五、與其他部門的協作

- **戰略部**：接收賽後檢討及補丁提案方向。
- **稽查部**：接收補丁合規審查結果。
- **會計部**：接收《財務結算表》進行歸檔。
- **中央治理委員會**：提交補丁審批。
