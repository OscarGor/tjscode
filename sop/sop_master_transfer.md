---

## 一、大師傳功的定義

大師傳功 = **賽前預測 → 賽後檢討 → 補丁沉澱 → 系統升級** 的完整閉環。

新任 AI 不需要從零開始學習賽馬或玄學，而是透過「執行 SOP → 比對賽果 → 找出失誤 → 寫入補丁」的方式，讓系統在每一場實戰中進化。

---

## 二、大師傳功五步流程
第一步：接收賽事 → 第二步：執行 SOP → 第三步：比對賽果 → 第四步：深度檢討 → 第五步：補丁沉澱

text

### 第一步：接收賽事
從大師（或使用者）接收賽事基本資料：
- 場地、路程、開賽時間
- 14匹馬的馬號、馬名、騎師、練馬師、檔位、賠率
- 近況表現（如有）

### 第二步：執行完整 SOP
嚴格按照 `sop/sop_master.md` 的九階段流程執行預測：
1. 階段0：劇本五層穿透
2. 階段1：梅花整體起卦
3. 階段2：奇門遁甲排盤
4. 階段3：每匹馬七術全覆蓋＋綜合評級
5. 階段4-6：稽查與審核
6. 階段7：戰略部終極仲裁＋$200 投注部署
7. 輸出預測結果

### 第三步：比對賽果
賽後接收賽果（名次及派彩），逐項比對：
- 哪些馬命中？哪些馬失誤？
- 獨贏、位置、連贏、三重彩、四重彩各命中多少？
- 計算回報率

### 第四步：深度檢討（核心）
對每一匹「唔中」的馬，必須追問：

| 問題 | 檢查方向 |
|------|----------|
| 梅花卦象是否解讀錯誤？ | 體用生剋、卦德、動爻位置、變卦組合 |
| 奇門格局是否忽略關鍵信號？ | 四害、十二長生、天干組合、八門八神 |
| 是否有補丁未觸發？ | 檢查 DP-001 至 DP-106 |
| 是否有新規律未被系統捕捉？ | 新的卦象組合、格局組合、檔位規律 |

### 第五步：補丁沉澱
將檢討中發現的新規律寫成補丁，格式如下：
補丁編號：DP-XXX
補丁名稱：[簡潔描述]
觸發條件：[何時觸發]
強制程序：[觸發後必須執行的步驟]
應用案例：[日期 場次 馬名 結果]

text

然後更新以下檔案：
- `rules/patches.md`：加入新補丁
- `data/hexagram_deep.md`：更新相關卦象的深化筆記
- `data/qimen_patterns.md`：更新相關格局的驗證記錄
- `data/archive/`：歸檔完整回測記錄

## 三、大師傳功的核心心法

| 心法 | 內容 |
|------|------|
| 每一場都是學習 | 無論贏輸，每一場都有教訓可以沉澱 |
| 補丁就是內力 | 每一條補丁都是大師的內力傳承，系統因此而壯大 |
| 梅花為尊 | 當多術共振時，梅花體用生剋是最終裁判 |
| 無四害就往好處想 | 奇門格局乾淨本身就是最大的吉象 |
| 雙重確認 | 玄學信號與數據信號正向一致時，可信度倍增 |
| 不怕錯，怕不記 | 每一次失誤都是補丁的原材料 |

## 四、新任 AI 的初始化步驟

當新任 AI 首次接手 TJSCODE 時，應按以下步驟完成初始化：

1. 讀取憲法：`CONSTITUTION.md`
2. 讀取總 SOP：`sop/sop_master.md`
3. 讀取補丁庫：`rules/patches.md`（106條補丁）
4. 讀取回測記錄：`data/archive/` 中至少三場完整回測
5. 讀取梅花庫：`data/hexagram_deep.md`
6. 讀取奇門庫：`data/qimen_patterns.md`
7. 讀取場地規則：`data/st_rules.md` 及 `data/hv_rules.md`
8. 讀取跨場次能量追蹤：`core/cross_energy.md`

完成以上八步後，新任 AI 即具備執行大師傳功的基礎能力。

## 七、交接備忘

新任 AI 請記住：

- 你不需要從零開始——106條補丁是前人用真金白銀換來的教訓。
- 每一場賽事都是大師傳功的機會——贏了沉澱經驗，輸了沉澱補丁。
- 系統的生命力來自補丁的持續進化——你的每一次檢討，都會讓 TJSCODE 變得更強。
- 古典凶格不可反轉（如青龍逃走），只能被現實條件壓制。
- 卦德（卦義）往往比體用生剋更重要。
